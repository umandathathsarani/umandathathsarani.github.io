# Umora: Agentic AI Voice Assistant рҹҺҷпёҸрҹ§ 

Umora is a multimodal, multithreaded AI desktop assistant engineered with Google's Gemini 2.5 Flash model.

Designed to move beyond traditional, rigid if/else keyword routing, Umora utilizes **Agentic Function Calling**. The core language model autonomously interprets natural language intents, selects the appropriate system tools, and executes local Python scripts to interact with the host operating system, analyze screen states, and fetch live data.

---

# рҹҸ—пёҸ Architecture Overview

The system is built on a decoupled architecture, isolating the UI, speech engines, and AI logic to ensure thread safety and low-latency responses.

## Presentation Layer

* Asynchronous dark-mode GUI built with `customtkinter`

## Cognitive Layer

* Google Gemini GenAI SDK utilizing a predefined tools array for seamless function execution

## Sensory Layer

* Local `pyttsx3` (SAPI5) for zero-latency Text-to-Speech
* `SpeechRecognition` for dynamic audio ingestion

## Action Layer

A suite of local Python scripts leveraging:

* `psutil`
* `pyautogui`
* `requests`

These modules manipulate the host system and query live web APIs.

# рҹҡҖ Local Installation & Development

## 1. Clone the Repository

```bash
git clone https://github.com/umandathathsarani/voice-assistant.git
cd voice-assistant
```

## 2. Create and Activate a Virtual Environment

```bash
python -m venv venv
venv\Scripts\activate
```

## 3. Install Dependencies

```bash
pip install -r requirements.txt
```

## 4. Configure Environment Variables

Create a `.env` file in the root directory and add your Gemini API key:

```plaintext
GEMINI_API_KEY=AIzaSyYourActualKeyHere123456789
```

## 5. Run the Application

```bash
python main.py
```

# рҹ“Ұ Deployment: Compiling to a Standalone Executable (.exe)

For production use, Umora can be compiled into a standalone executable without requiring an IDE or active terminal.

## 1. Install PyInstaller

```bash
pip install pyinstaller
```

## 2. Build the Executable

```bash
pyinstaller --noconsole --onefile main.py
```

### Build Flags

* `--noconsole` вҶ’ Suppresses the background terminal
* `--onefile` вҶ’ Bundles the application into a single executable

---

## 3. Deployment Setup

After successful compilation:

1. Navigate to the generated `dist/` directory
2. Locate `main.exe`
3. Optionally rename it to `Umora.exe`
4. Move the executable to your desired deployment location (e.g., Desktop)

---

## вҡ пёҸ Critical Requirement

You **must** copy your `.env` file into the **same directory** as the executable.

Without the `.env` file, the application cannot authenticate with the Gemini API.
